# План міграції Диспетчера (Mobile -> Web)

## 1. Бекенд (Django API & Admin)
Оскільки бекенд вже є (Django Admin), ми можемо швидко зробити повноцінну панель диспетчера на базі Django або окремого SPA-додатку.

**Завдання:**
1. Створити нові або розширити існуючі API-ендпоінти для веб-клієнта (якщо це окремий SPA React/Vue).
2. АБО кастомізувати існуючий Django Admin:
   - Створити окрему групу "Dispatchers" з доступом тільки до потрібних моделей (Замовлення, Користувачі, Авто, Відгуки).
   - Зробити кастомні Admin-сторінки з картою та зручним інтерфейсом для створення замовлень.
3. Додати функціонал:
   - **Карта в реальному часі** для моніторингу всіх активних водіїв та замовлень.
   - **Облік автомобілів** (додавати, редагувати авто, закріплювати їх за водіями).
   - **Статистика** (кількість поїздок, дохід системи).
   - Керування "Скарги" (resolving скарг від пасажирів).

## 2. Веб-інтерфейс (Frontend)
Розробка окремого диспетчерського дашборду. Залишаємо фіолетову тему та стилістику CLIX (стиль схожий на додаток, але адаптований під мишку і великий екран).

**Логічні блоки UI:**
- **[A] Бокова панель (Sidebar):**
  - Дашборд (карта + загальна статистика).
  - Замовлення (список, фільтри по статусах).
  - Водії та Авто (облік, онлайн/офлайн статус, рейтинг).
  - Пасажири.
  - Скарги та Відгуки.
- **[B] Основна робоча зона (Дашборд):**
  - Велика карта з маркерами (вільні водії, активні поїздки).
  - Панель швидкого створення замовлення (форма з пошуком адрес, розрахунком ціни, вибором типу авто).
- **[C] Робота з водіями (Автопарк):**
  - Таблиця: Водій -> Машина -> Номер -> Клас -> Статус.
  - Можливість блокувати водія (якщо він порушує правила).

## 3. Мобільний додаток (Очищення)
- Видалити `dispatcher_home_screen.dart` та пов'язнані з ним UI файли.
- Прибрати роль `DISPATCHER` з RoleSelector (тепер диспетчери не логіняться з телефону).
- Очистити в `ApiService` старі ендпоінти `dispatcher/orders/` і т.д. (залишити їх тільки для веба).

## 4. Етапність реалізації
1. **Етап 1 (Бекенд):** Створення моделей (якщо бракує, напр. детальний прив'язок Авто-Водій), пермішенів для веб-доступу. Реалізація ендпоінтів для загальної карти (всі машини).
2. **Етап 2 (Веб UI):** Верстка Sidebar, таблиць Замовлень/Водіїв/Автомобілів, форма створення нового замовлення телефоном через диспетчера.
3. **Етап 3 (Live Map):** Інтеграція Leaflet/Google Maps у веб для відображення всіх машин.
4. **Етап 4 (Очищення):** Видалення непотрібного коду з Flutter-додатку.
