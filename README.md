# 🚖 CLIX — Таксі в один клік

CLIX — це універсальна платформа для замовлення та диспетчеризації таксі. Проєкт розроблено як повноцінну систему, що складається з потужного бекенду на Django та кросплатформного клієнтського додатку на Flutter (iOS, Android, macOS).

## ✨ Особливості

- **Три ролі в одному додатку**: Пасажир, Водій, Диспетчер. Додаток автоматично адаптує інтерфейс під обрану роль або дає змогу швидко перемикатись між ними.
- **Жива карта та відстеження**: Інтеграція `flutter_map` та `geolocator` для відстеження поїздок та місцезнаходження водіїв у реальному часі (з автоматичним розрахунком ETA).
- **Оцінка вартості**: Автоматичний розрахунок вартості поїздки на бекенді з використанням формули Гаверсина (Haversine formula), що враховує відстань та обраний клас авто (Економ, Комфорт, Бізнес, Мінівен).
- **Сучасний UI/UX**: Динамічні інтерфейси на основі Material 3 з підтримкою спеціальних тем: світлої для пасажира та комфортної темної для водія в дорозі.
- **Відгуки та рейтинг**: Можливість оцінювати водіїв та керувати репутацією.

## 🏗 Архітектура (Технічний Стек)

### Бекенд (Django API)

- **Фреймворк**: Django 6.0 + Django REST Framework (DRF)
- **База даних**: SQLite (для локальної розробки, повністю підтримує міграцію на PostgreSQL у продакшені)
- **Авторизація**: JWT (JSON Web Tokens) через SimpleJWT
- **Основні модулі**:
  - `accounts` — Моделі користувачів, менеджмент сесій та ролей.
  - `orders` — Логіка замовлень, призначення водіїв, життєвий цикл поїздки та відгуки.
  - `vehicles` — Транспортні засоби, параметри (дитяче крісло, pet friendly) та їх класифікація.

### Фронтенд (Flutter App)

- **Фреймворк**: Flutter (Dart ^3.x)
- **Стейт-менеджмент**: `provider`
- **Мережа**: `dio` для роботи з REST API
- **Безпека**: `flutter_secure_storage` для надійного збереження токенів.
- **Картографія**: `flutter_map` (тайли OpenStreetMap, CartoDB Voyager/DarkMatter), `geolocator` для позиціонування.

## 🚀 Встановлення та запуск (Локально)

### 1. Запуск Бекенду (Django)

Відкрийте термінал та перейдіть у кореневу папку проєкту:

```bash
# Активуйте віртуальне середовище
source venv/bin/activate

# Встановіть залежності (якщо нове розгортання)
# pip install -r requirements.txt

# Зробіть міграції бази даних
python manage.py migrate

# Запустіть сервер
python manage.py runserver
```

API буде доступне за базовою адресою: `http://127.0.0.1:8000/api/`

### 2. Запуск Додатку (Flutter)

Відкрийте нову вкладку терміналу та перейдіть у каталог додатка:

```bash
cd clix_app

# Встановіть залежності
flutter pub get

# Запуск додатку на macOS (можна також використовувати iOS/Android симулятори)
flutter run -d macos
```

## 📂 Дерево проєкту

```text
Diploma/
├── manage.py            # Вхідна точка Django
├── db.sqlite3           # Локальна база даних
├── clix_backend/        # Налаштування та конфіги Django (settings.py)
├── accounts/            # Додаток Django: Користувачі, ролі та авторизація
├── orders/              # Додаток Django: Бізнес-логіка замовлень
├── vehicles/            # Додаток Django: Автомобілі та їх характеристики
└── clix_app/            # Клієнтський додаток на Flutter
    ├── lib/
    │   ├── main.dart    # Вхідна точка Flutter
    │   ├── config/      # Теми, Константи, URL конфігурація
    │   ├── models/      # Data Classes
    │   ├── providers/   # Стейт-менеджери (напр., AuthProvider)
    │   ├── screens/     # UI екрани (розділені по ролях та auth)
    │   ├── services/    # Бізнес логіка фронтенду (API, Pricing, Geo)
    │   └── widgets/     # Спільні компоненти та віджети (кнопки, інпути)
    ├── assets/          # Зображення, іконки додатку та шрифти
    ├── pubspec.yaml     # Залежності Flutter
    └── flutter_launcher_icons.yaml # Конфігуратор іконок додатку
```
